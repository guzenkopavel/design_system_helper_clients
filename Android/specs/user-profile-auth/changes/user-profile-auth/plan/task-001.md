## Goal

Создать изолированный Gradle Android library модуль для возможности авторизации,
подключить его к обнаруженному графу проекта, определить минимальный публичный
контракт компонуемого шлюза и замкнутого класса состояния, сохранив модуль
приложения в роли только композиции и маршрутизации.

## Inline contract context

`AND-REQ-1` требует изолированную границу библиотеки. `AND-REQ-3` требует, чтобы
модуль приложения содержал только точку входа и корневую маршрутизацию.
`AND-AC-1` требует собираемую границу и одностороннюю композицию. `AND-AC-4`
требует граф сборки с модулем библиотеки авторизации, где модуль приложения
зависит от контракта и не содержит области авторизации, данных или изменяемого
состояния.

## Implementation deliverables

- Появится Gradle Android library модуль с файлом сборки, манифестом и минимальной
  структурой исходного кода, подключённый к графу проекта; модуль собирается
  независимо и раскрывает публичные типы компонуемого шлюза и замкнутого класса
  состояния для корневой маршрутизации приложения.
- Публичный контракт модуля формализует однонаправленную зависимость: приложение
  зависит от модуля авторизации, а модуль авторизации не зависит от приложения и
  оболочки; защитник границы гарантирует, что логика авторизации, изменяемое
  состояние и данные остаются за границей модуля и недоступны потребителю.

## Steps

Добавить включение модуля авторизации в файл настроек сборки, создать Gradle
Android library файл сборки на обнаруженных каталоговых псевдонимах, добавить
минимальный манифест и структуру исходного кода. Создать файлы заполнителей для
шлюза с пустым компонуемым элементом и замкнутого класса состояния для тестов
нижних уровней. Поддерживать границу модуля явной: наружу виден только
минимальный публичный контракт, модуль приложения не владеет состоянием
возможности, а граф зависимостей остаётся ациклическим. После появления модуля
обновить поиск задач сборки и записать новые имена задач сборки модуля перед
использованием.

Граф проекта содержит собираемый изолированный модуль библиотеки, цикл зависимостей
