# Android addendum: Reconcile Implementation

Use the Android adapter production roots and protected/excluded roots. Discover
the current Gradle modules, variants, tasks and focused test command from
repository/package evidence; never invent them. Compare Kotlin/Compose and
Gradle changes against the package's selected Android architecture, coroutine,
testing, accessibility and design-system rules only when those scopes apply.

Evidence must record the exact finite focused command and result for the
explicit Android paths. Reconciliation remains package-only: it cannot edit
Kotlin/Gradle production files, shared product artifacts or iOS state.
