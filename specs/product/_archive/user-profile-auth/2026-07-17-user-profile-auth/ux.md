# user-profile-auth — shared product UX

- **Applies to:** `iOS, Android`
- **UX review:** `PENDING`

## User Journey

Пользователь запускает приложение. Без валидной сессии он попадает в тихий
двухшаговый флоу авторизации: сначала вводит почту, затем — пароль на шаге,
явно озаглавленном «Вход» либо «Регистрация» в зависимости от того, существует
ли аккаунт. Успешный шаг пароля немедленно открывает знакомую оболочку с
тремя разделами и стартовым разделом «Кейсы». При повторных запусках в
пределах жизни сессии пользователь попадает в оболочку сразу, минуя флоу.
Когда сессия истекает или отзывается, пользователь возвращается на шаг почты
с коротким объяснением причины.

## Entry and Exit Points

- Вход в флоу: запуск приложения без валидной сессии; возврат по истечению
  или отзыву сессии (ответ 401 на персональный запрос).
- Выход из флоу: успешный вход или регистрация → оболочка, раздел «Кейсы».
- Обратного входа в флоу из оболочки по инициативе пользователя в этой фиче
  нет: действие «Выйти» вынесено в будущую фичу профиля.

## Flow and Screen Impact

| Step | User intent | Surface/screen | Observable result | iOS/Android variance |
|---|---|---|---|---|
| Запуск | Открыть приложение | Стартовое состояние определения сессии | Спокойный нейтральный экран загрузки без мигания формой входа; затем оболочка либо шаг почты | Различий нет |
| Шаг почты | Назвать себя | Экран авторизации, шаг «Почта» | Поле почты, одно основное действие «Продолжить»; отправка запускает проверку с видимым индикатором | Нативная клавиатура почтового типа |
| Ветвление | Понять следующий шаг | Экран авторизации, шаг «Пароль» | Заголовок «Вход» для существующей почты либо «Регистрация» для новой; введённая почта остаётся видимой | Различий нет |
| Шаг пароля | Войти или создать аккаунт | Экран авторизации, шаг «Пароль» | Скрытое поле пароля с управляемым раскрытием, одно основное действие; ошибка показывается рядом с формой | Нативный контроль раскрытия пароля |
| Возврат | Исправить почту | Шаг «Пароль» → шаг «Почта» | Явное действие возврата; введённая почта сохранена и доступна для правки | Системный back (Android) и навигационный возврат (iOS) ведут себя одинаково с явным действием возврата |
| Успешный вход | Начать работу | Оболочка `app-shell` | Оболочка с тремя разделами, активны «Кейсы»; поверхности остаются baseline-заглушками | Различий нет |
| Возврат по сессии | Продолжить работу после перерыва | Любой экран → шаг почты | Короткое объяснение, что сессия истекла и нужно войти снова; данные прошлой сессии не видны | Различий нет |

## States

- **Happy path:** почта → проверка → шаг пароля с корректным заголовком →
  успех → оболочка. Повторный запуск в пределах жизни сессии → оболочка сразу.
- **Loading:** определение сессии при запуске; проверка почты; отправка
  входа/регистрации. Во время запроса основное действие заблокировано и
  показан индикатор; повторная отправка невозможна.
- **Empty:** пустая почта или пустой пароль не отправляются; попытка
  продолжить показывает человекочитаемое объяснение рядом с полем, основное
  действие остаётся видимым и доступным для активации.
- **Error (валидация):** синтаксически невалидная почта и пароль короче 6
  символов при регистрации отклоняются до сетевого запроса с человекочитаемым
  сообщением на русском.
- **Error (сервер):** неверные учётные данные сообщаются без раскрытия,
  существует ли аккаунт; поле пароля очищается, повтор доступен сразу. Занятая
  почта на шаге регистрации объясняет, что аккаунт уже есть, и предлагает
  выполнить вход.
- **Rate limit:** отдельное состояние «попытки временно ограничены» с
  указанием времени, когда можно повторить; до истечения срока основное
  действие не отправляет запрос. Появление этого состояния объявляется
  ассистивными технологиями наравне с ошибками.
- **Offline:** отсутствие сети — отдельное восстановимое состояние с
  возможностью повторить после восстановления связи; введённые данные, кроме
  пароля, сохраняются.
- **Recovery:** истёкшая или отозванная сессия возвращает на шаг почты с
  коротким объяснением; недействительный сессионный секрет очищается из
  хранилища устройства; после повторного входа пользователь снова в оболочке.

## Navigation

- Флоу линеен: шаг почты ↔ шаг пароля; вглубь навигации нет.
- Возврат с шага пароля на шаг почты — явное действие, продублированное
  системным жестом/кнопкой возврата платформы; почта при возврате сохранена.
- На шаге почты системный возврат ведёт себя по-платформенному стандартно
  (осознанная platform variance); флоу не перехватывает его нестандартно.
- После успеха флоу закрывается целиком; вернуться в него жестом «назад»
  из оболочки нельзя.

## Content and Interaction Semantics

- Тексты — на русском, спокойные и краткие: без промо-контента, без
  технических кодов в основном тексте сообщения.
- Заголовок шага пароля явно называет режим: «Вход» либо «Регистрация»;
  пользователь никогда не угадывает ветку сам.
- Ошибки появляются рядом с формой и не заменяются немым сбросом состояния.
- Ровно одно основное действие на каждом шаге; во время запроса оно
  заблокировано с видимым индикатором.
- Поле пароля скрывает ввод; раскрытие видимости — явное управляемое действие.
- Системное автозаполнение почты и пароля допускается как нативная
  возможность платформы; флоу не зависит от его наличия.

## Accessibility and Localization

- Доступные имена полей и действий совпадают с видимыми русскими подписями.
- Появление ошибки, состояния ограничения попыток и начало/конец загрузки
  объявляются ассистивными технологиями; заблокированность действия
  передаётся ассистивно.
- Смена шага переводит фокус к логическому началу нового шага.
- Все состояния различимы без опоры только на цвет: ошибки, выбранность и
  заблокированность имеют текстовые или иконографические признаки.
- Поддержаны увеличенный текст, светлое, тёмное и повышенно-контрастное
  оформление; формы остаются читаемыми и работоспособными.
- Тексты фиксированы на русском текущим контрактом; будущая локализация не
  меняет структуру и порядок шагов.

## Analytics and Privacy

- Аналитика не добавляется: событий и свойств нет, измерений нет.
- Пароль не сохраняется, не логируется и не отображается после отправки.
- Постоянно видимая персональная информация — только почта в пределах флоу;
  оболочка персональных данных не показывает.
- Сообщения об ошибках могут содержать идентификатор трассировки для
  поддержки, но не раскрывают существование аккаунта сверх ветвления
  «Вход»/«Регистрация», которое уже сообщает сервер.

## Shared Design-System Intent

Default visual direction: спокойный, мягко-синий через semantic roles поверх
тихих нейтральных поверхностей. Отклонений от soft blue нет.

### Component roles

- Роль «поле формы»: ввод почты и пароля с подписью, состоянием ошибки и
  ассистивным именем.
- Роль «основное действие»: одна кнопка продолжения/отправки на шаг,
  использующая primary-акцент.
- Роль «второстепенное действие»: возврат к шагу почты, раскрытие пароля.
- Роль «сообщение обратной связи»: контейнер ошибки/ограничения попыток с
  семантикой критической обратной связи, а не только цветом.
- Роль «индикатор ожидания»: состояние загрузки шага и заблокированного
  действия.

### Token and semantic intent

- Primary action и focus — мягко-синий семантический акцент.
- Критическая обратная связь (ошибки) — семантическая роль ошибки с
  достаточным контрастом; не единственный носитель смысла.
- Поверхности — тихие нейтральные; иерархия достигается типографикой и
  интервалами, а не насыщенными заливками.
- Заблокированность действия выражается семантическим состоянием, а не только
  снижением непрозрачности.

### Visual language and appearance

- Мягко-синий закреплён за: основным действием, фокусом ввода, выделением
  активного шага и информационным акцентом объяснений.
- Светлое, тёмное и повышенно-контрастное оформления сохраняют читаемость
  подписей, сообщений и состояний полей.
- Не-цветовые признаки: текст ошибки рядом с полем, иконография состояния,
  явные подписи режима «Вход»/«Регистрация», индикатор загрузки.

### Allowed platform variance

- Нативные клавиатуры, контроль раскрытия пароля, системные жесты возврата и
  системное автозаполнение — платформенные, при сохранении порядка шагов,
  текстов и семантики.
- Поведение системного возврата на первом шаге — стандартное для платформы
  (Android может закрыть приложение; iOS не имеет системного выхода) — принятая
  variance, не меняющая product intent.
- Форма индикатора загрузки и диалоговых поверхностей — нативная.
