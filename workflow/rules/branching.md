# Branching and integration

Ветки и интеграция входят только в выбранный `delivery` scope и требуют явной
авторизации пользователя. Harness не предполагает имя основной ветки, hosting
provider или merge strategy: они извлекаются из repository settings/docs.

- Создавать ветку только когда это часть порученного delivery.
- Имя отражает один bounded change и следует обнаруженной convention.
- Не переписывать опубликованную или общую историю без согласования владельцев.
- Конфликты разрешать после понимания обеих сторон; после разрешения запускать
  затронутые проверки и искать оставшиеся conflict markers.
- Merge/rebase/squash выбираются по политике репозитория. При отсутствии
  политики остановиться перед необратимой интеграцией и спросить человека.
- Удаление remote/local branches — отдельное изменение состояния и не
  выполняется автоматически.

Branch isolation не даёт права коммитить пользовательские файлы. При совместной
работе ownership определяется change scope и текущим worktree status.
