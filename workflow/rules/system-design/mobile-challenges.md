# Mobile challenges

В proposal/design отметить `applicable`/`not applicable` с причиной для:

- state restoration и lifecycle interruptions;
- offline/cache/sync/conflict policy;
- deeplinks, push и launch routing;
- modular boundaries и dependency injection;
- feature flags, migration и rollback;
- performance: launch, memory, network, rendering, battery;
- accessibility и localization;
- analytics/privacy/crash observability;
- cross-client parity без общего implementation design;
- binary distribution и store review.

Применимость не означает автоматическое включение в scope, но скрытый пропуск
запрещён.
