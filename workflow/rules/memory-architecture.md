# Memory architecture

## SSOT

Каждое правило или процедура должны иметь одного канонического владельца.
Остальные файлы ссылаются на него и добавляют только локальный контекст.

## Канон и адаптер

- `workflow/` хранит переносимое процессное знание.
- `<platform>/` хранит платформенные расширения общего процесса.
- `.agents/skills/` хранит portable skill SSOT, общий для совместимых runtime.
- `.codex/`, `.claude/`, `.cursor/` и `.opencode/` хранят только runtime
  binding: команды, invocation policy, имена ролей, permissions, доступные
  инструменты и короткие указатели на канон.

Если адаптер начинает пересказывать процедуру, перенести содержание в канон и
оставить ссылку с runtime-специфичным дополнением.

Runtime adapter может повторить только обязательную метаинформацию обнаружения
(`name`, `description`, command name). Процедурные шаги должны оставаться в
portable skill или `workflow/`.
