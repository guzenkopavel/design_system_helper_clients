# Shared visual language

The project default is a calm, soft-blue visual direction expressed through
semantic roles, never scattered hexadecimal values. Shared product UX owns the
mood, hierarchy, emphasis, feedback and allowed variance; it must not name
platform design languages, APIs, SDK types or platform components.

Quiet neutral surfaces support the blue direction. Accessibility, contrast and
non-color cues dominate brand mood: every applicable state must remain legible
and distinguishable in light, dark and high/increased-contrast appearances.
Platform implementations adapt the intent through native design languages and
existing design-system roles. Any deviation from soft blue requires explicit
product rationale in the shared UX package.

Shared UX specifies semantic roles such as primary action, selection, focus,
informational accent and critical feedback. Platform UX discovers real tokens,
components, deployment/SDK availability and maps those roles without inventing
API support or copying literal color values into feature code.
