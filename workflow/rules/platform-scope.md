# Platform scope

Перед изменением определить один scope:

| Scope | Каноническое размещение | Обязательная проверка |
|---|---|---|
| `common` | `workflow/`, `process/`, root adapters | общая структура |
| `ios` | `iOS/` | iOS-срез |
| `android` | `Android/` | Android-срез |
| `cross-platform` | общий контракт + тонкие платформенные расширения | iOS и Android независимо |

Не копировать одинаковое правило в обе платформы. Общую часть поднять в
`workflow/`, а в платформенных каталогах оставить только различия.

Для cross-platform изменения отчёт должен содержать отдельные evidence-блоки
`iOS` и `Android`. Отсутствие реализации платформы является явным blocker или
зафиксированным non-goal, но не молчаливым успехом.
