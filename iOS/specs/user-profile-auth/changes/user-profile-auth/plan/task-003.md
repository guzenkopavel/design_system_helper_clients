## Goal

Реализовать хранилище c явным классом защиты и атрибутами доступности.
Создать тестовую реализацию для модульных проверок. Тесты подтверждают
чтение, запись, удаление и отсутствие секрета после очистки.

## Inline contract context

- `REQ-6` — истечение или отзыв сессии очищает недействительный секрет из хранилища.
- `REQ-7` — пароль не сохраняется, сессионные данные хранятся недоступно другим приложениям.
- `AC-16` — после возврата недействительный секрет отсутствует в хранилище.
- `AC-17` — после входа сохранённый пароль отсутствует.
- `AC-18` — сессионные данные недоступны другим приложениям.
- `IOS-REQ-9` — значение хранится в защищённом хранилище c явным классом доступности, без групп; недоступно другим приложениям.
- `IOS-REQ-8` — недействительный секрет удаляется при возврате.
- `AC-16` — после возврата на экран входа по истечению или отзыву сессии недействительный секрет отсутствует в хранилище.
- `AC-17` — после входа сохранённый пароль отсутствует.

## Implementation deliverables

- Реализация хранилища, соответствующая протоколу, c операциями чтения, записи и удаления через системную инфраструктуру безопасности c явным атрибутом доступности и без групп.
- Тестовая реализация c тем же протоколом, используемая модульными тестами вместо реального хранилища для детерминированности и независимости от состояния устройства.
- Тесты хранилища, подтверждающие запись и чтение, удаление при очистке, отсутствие секрета после удаления и идемпотентность повторного удаления без ошибки.

## Steps

1. Создать файл реализации хранилища.
2. Реализовать операции через системную инфраструктуру безопасности c явным атрибутом доступности.
3. Обработать ошибки: отсутствие элемента при чтении возвращает пустое значение, а не ошибку.
4. Создать тестовую реализацию в памяти.
5. Написать тесты: чтение пустого, запись, чтение, удаление, идемпотентность.
6. Запустить тесты c watchdog 300 секунд.

## Verification

- Все тесты хранилища проходят c тестовой реализацией.
- Реализация собирается без предупреждений.
- `application boundary` — хранилище не импортирует интерфейс.
- `package consumer` — пакет собирается c хранилищем.
- `consumer integration` — тестовая цель подключена.
- `package build` — сборка успешна.
- `public contract` — хранилище закрыто за протоколом.
- `dependency graph` — направление от данных к домену.
- `app-shell wiring` — хранилище не знает о потребителе.
