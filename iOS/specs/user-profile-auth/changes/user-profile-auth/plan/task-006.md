## Goal

Реализовать поверхность флоу c шагами почты и пароля, обратной связью,
загрузкой и нативной семантикой. Применить условное оформление навигации
c запасным вариантом для старых версий. Обеспечить вспомогательную
семантику, масштабирование текста, варианты оформления и совпадение
видимых и вспомогательных имён c русскими подписями.

## Inline contract context

- `REQ-2` — флоу начинается c ввода почты, ветвление и возврат без потери значения.
- `REQ-5` — ошибки флоу наблюдаемы и восстановимы через представление.
- `REQ-8` — флоу доступен ассистивно и в согласованных вариантах оформления.
- `REQ-9` — наблюдаемое поведение, тексты шагов и правила совпадают c общим контрактом.
- `AC-19` — ассистивные имена совпадают с подписями.
- `AC-20` — фокус переходит к началу нового шага.
- `AC-22` — формы читаемы при увеличенном тексте.
- `AC-23` — светлое и тёмное оформление сохраняют читаемость.
- `IOS-REQ-2` — стандартные контролы; русские подписи совпадают c вспомогательными именами; масштабирование текста; варианты оформления.
- `IOS-REQ-5` — шаг почты c действием продолжения; шаг пароля c заголовком входа или регистрации; возврат c сохранённой почтой; раскрытие пароля.
- `IOS-REQ-7` — ошибки рядом c полем; загрузка блокирует действие; ограничение c временем повтора; отсутствие сети c повтором.
- `IOS-REQ-13` — тексты совпадают c общим контрактом.
- `AC-19` — ассистивные имена совпадают c видимыми подписями, состояния объявляются.
- `AC-22` — формы читаемы и работоспособны при увеличенном тексте.

## Implementation deliverables

- Открытый вид флоу, принимающий модель представления и отображающий текущий шаг c навигационным контейнером, программным переходом к шагу пароля и системным возвратом, эквивалентным действию возврата.
- Вид шага почты c текстовым полем почты c клавиатурой почты, основным действием продолжения c семантическим акцентом, индикатором загрузки и сообщением ошибки рядом c полем, совпадающим c вспомогательным именем.
- Вид шага пароля c заголовком входа или регистрации в зависимости от режима, видимой почтой, скрытым полем пароля c управляемым раскрытием, основным действием входа или регистрации и навигационным возвратом к шагу почты.
- Вид обратной связи c семантической ролью критической обратной связи, текстом и изобразительным признаком ошибки, ограничения попыток c обратным отсчётом и состояния отсутствия сети c действием повтора, различимый не только цветом.
- Условное применение оформления навигации c проверкой доступности и стандартным непрозрачным запасным вариантом на старых версиях и при недоступности, без собственной имитации размытия или материалов.

## Steps

1. Создать вид флоу c навигационным контейнером и программным переходом по шагу.
2. Создать вид шага почты: поле c клавиатурой, кнопка продолжения, вид обратной связи для ошибок.
3. Создать вид шага пароля: заголовок по режиму, видимая почта, скрытое поле c раскрытием, кнопка входа или регистрации.
4. Создать вид обратной связи c семантической ролью, текстом и значком.
5. Создать наложение загрузки c индикатором и блокировкой действия.
6. Применить условное оформление: проверка доступности c запасным стандартным оформлением.
7. Добавить идентификаторы вспомогательных технологий; убедиться, что имена совпадают c видимыми подписями.
8. Проверить масштабирование текста c адаптивной вёрсткой и прокруткой.
9. Проверить варианты оформления визуально на симуляторе.
10. Запустить сборку c watchdog 300 секунд.
