## Goal

Реализовать конечный автомат флоу c семантикой значения, типизированными
действиями и политикой единственной активной задачи. Локальная проверка
выполняется до сетевого запроса. Тесты подтверждают все переходы,
конкурентность, отмену и отображение ошибок.

## Inline contract context

- `REQ-2` — флоу начинается c ввода почты, ветвление и возврат без потери значения.
- `REQ-5` — ошибки флоу наблюдаемы и восстановимы через автомат состояний.
- `AC-2` — почта существующего аккаунта ведёт к шагу с заголовком «Вход».
- `AC-3` — почта без аккаунта ведёт к шагу с заголовком «Регистрация».
- `AC-4` — возврат к шагу почты сохраняет введённую почту.
- `AC-8` — неверный пароль показывает восстановимую ошибку.
- `AC-12` — ограничение попыток показывает время повтора.
- `IOS-REQ-5` — флоу начинается c шага почты; отправка почты — проверка до сети; результат ветвит режим входа или регистрации.
- `IOS-REQ-7` — пустые поля, невалидная почта, короткий пароль, ошибки сервера, ограничение попыток, отсутствие сети — все наблюдаемы.
- `IOS-REQ-11` — единственная активная задача; повторная активация игнорируется; уход отменяет запрос; отмена не меняет состояние.
- `AC-8` — неверный пароль показывает восстановимую ошибку без раскрытия существования аккаунта.
- `AC-12` — ответ об ограничении попыток переводит флоу в состояние c указанием времени повтора.

## Implementation deliverables

- Семантическое значение состояния c полями шага почты или пароля c режимом и подтверждённой почтой, фазы ожидания или загрузки и обратной связи c вариантами ошибок проверки, учётных данных, занятости, проверки сервера, ограничения попыток, отсутствия сети и истечения.
- Перечисление действий c вариантами изменения почты, пароля, отправки почты, отправки пароля, переключения видимости, возврата и повтора, каждое из которых обрабатывается единственным методом модели.
- Реализация модели c изоляцией главного действующего лица и наблюдаемостью, владеющая ровно одной структурированной задачей на шаг c монотонным счётчиком поколения, проверяющей совпадение после каждого ожидания для исключения поздних изменений от отменённых запросов.
- Локальная проверка почты и пароля до сетевого запроса c возвращаемыми сообщениями рядом c полем, без вызова клиента при невалидных данных.
- Обратный отсчёт ограничения c инжектированным источником времени, блокирующий отправку до наступления срока и отменяющийся при уходе состояния.
- Тесты модели, покрывающие все переходы автомата, локальную проверку, отображение серверных ошибок, политику единственной отправки, отмену при возврате и отсутствие поздних изменений.

## Steps

1. Создать файл состояния c шагами, фазами, обратной связью и вложенными типами.
2. Создать файл действий.
3. Реализовать модель c изоляцией, наблюдаемостью, состоянием, методом отправки, активной задачей и поколением.
4. Локальная проверка в отправке почты и пароля.
5. Политика единственной отправки: при загрузке — игнорировать; при новом действии — отменить предыдущую задачу.
6. После ожидания — проверить совпадение поколения; при несовпадении — не менять состояние.
7. Возврат — отмена задачи, возврат к шагу почты c сохранённой почтой, очистка пароля.
8. Написать подделки вариантов использования и источника времени.
9. Написать тесты: пустая почта не вызывает клиент; все переходы; ошибки; отмена; поколение защищает от поздних изменений.
10. Запустить тесты c watchdog 300 секунд.

## Verification

- Все тесты модели проходят.
- `cancellation` — уход отменяет задачу, отмена не меняет состояние.
- `isolation` — изоляция главного действующего лица, передаваемые границы, нет отсоединённых задач.
- `simulator` — модель собирается для симулятора.
- `accessibility` — состояния наблюдаемы для вспомогательных технологий.
- `design-system` — семантические роли в обратной связи.
- `platform-ux.md` — состояния соответствуют карте интерфейса.
- `Liquid Glass` — не применяется на этом слое.
- `light/dark` — состояния не зависят от оформления.
- `increased contrast` — состояния различимы.
- `Reduce Transparency` — не применяется.
- `Reduce Motion` — не применяется.
- `older-OS/SDK fallback` — модель не зависит от версии.
