## Goal

Реализовать фабрику c точками сборки рабочего графа и заглушки для тестов,
а также модель сессии — наблюдаемый владелец состояния c идемпотентной
проверкой при запуске. Тесты подтверждают минимальность открытого контракта
и корректность переходов сессии.

## Inline contract context

- `REQ-1` — приложение целиком за авторизацией, фабрика и модель обеспечивают композицию.
- `REQ-4` — проверка сессии при запуске, хранение между запусками.
- `REQ-6` — истечение или отзыв сессии очищает секрет и возвращает на вход.
- `AC-5` — верные учётные данные открывают оболочку.
- `AC-7` — перезапуск в пределах жизни сессии открывает оболочку без входа.
- `AC-15` — истечение сессии возвращает на вход.
- `IOS-REQ-4` — запуск идемпотентен; проверка сессии при запуске; переход в активность или выход.
- `IOS-REQ-8` — недействительность персонального запроса — очистка секрета и выход c причиной.
- `IOS-REQ-12` — открытый контракт минимален: конфигурация, состояние, модель, вид и фабрика.
- `AC-4` — запуск без валидной сессии показывает спокойное состояние загрузки, затем шаг почты.
- `AC-5` — перезапуск в пределах жизни сессии открывает оболочку сразу, без экрана входа.

## Implementation deliverables

- Открытая фабрика c методами сборки рабочего графа c клиентом, хранилищем и вариантами использования, и заглушки для детерминированных тестов c настраиваемым сценарием.
- Модель сессии c изоляцией главного действующего лица и наблюдаемостью c состоянием, идемпотентным методом запуска, выполняющим проверку ровно один раз, и переходами по успеху флоу и недействительности сессии.
- Тесты фабрики, подтверждающие сборку рабочего графа и заглушки, а также тесты модели, покрывающие идемпотентность запуска, переход в активность при валидной сессии и возврат в выход c причиной при недействительности.

## Steps

1. Создать модель сессии c состоянием, запуском, входом и недействительностью.
2. Идемпотентность запуска: флаг, повторный вызов — без действия.
3. Проверка сессии: при успехе — активность; при недействительности — очистка и выход c причиной; при отсутствии сети — выход без причины.
4. Создать фабрику: сборка рабочего графа c рабочими зависимостями; заглушка c подделками.
5. Убедиться, что все остальные типы закрыты.
6. Написать тесты: запуск без секрета — выход; идемпотентность; активность; недействительность.
7. Запустить тесты c watchdog 300 секунд.
