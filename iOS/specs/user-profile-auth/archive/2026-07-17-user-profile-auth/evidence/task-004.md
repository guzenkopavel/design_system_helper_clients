# Итог

Реализованы четыре варианта использования: проверка почты, вход, регистрация и проверка сессии. Каждый вариант делегирует сетевому клиенту через протокол и возвращает доменные типы. Проверка сессии маршрутизирует недействительность в очистку хранилища. Вход и регистрация сохраняют сессионный секрет.

# Проверки

Команда тестирования пакета: 25 тестов, 0 провалов.

- Проверка почты делегирует клиенту и возвращает результат — прошёл
- Проверка почты возвращает ложное значение — прошёл
- Проверка почты пробрасывает ошибку — прошёл
- Вход сохраняет сессионный секрет — прошёл
- Вход пробрасывает ошибку — прошёл
- Регистрация сохраняет сессионный секрет — прошёл
- Регистрация пробрасывает ошибку — прошёл
- Проверка сессии без токена возвращает ложное значение — прошёл
- Проверка сессии с валидным токеном возвращает истинное значение — прошёл
- Проверка сессии при недействительности очищает хранилище — прошёл
- Проверка сессии пробрасывает другие ошибки — прошёл

- Изоляция: варианты передаваемы, не удерживают мутабельное состояние
- Граница приложения: варианты не импортируют интерфейс и транспорт
- Потребитель пакета: пакет собирается с вариантами
- Интеграция потребителя: тестовая цель подключена
- Сборка пакета: успешна
- Публичный контракт: варианты закрыты за протоколами
- Граф зависимостей: направление от домена к деталям
- Подключение оболочки: варианты не знают о потребителе
